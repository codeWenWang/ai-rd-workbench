# 项目分析折叠导航设计

## 目标

将侧边栏中的“架构图”“流程图”“时序图”“接口文档”归入“项目分析”一级分组，减少默认展开的导航项数量。

## 交互

- “项目分析”使用可点击的一级导航按钮，并显示方向箭头。
- 页面首次加载或刷新时，项目分析分组默认收起，不持久化上一次展开状态。
- 点击一级按钮后切换展开和收起，四个子项使用短暂的高度与透明度过渡。
- 进入分析子页面后，对应子项继续保持选中状态；用户仍可手动收起分组。
- 整体侧边栏收起时不显示分析子项；移动端侧边栏沿用相同分组行为。

## 实现范围

- 调整 `frontend/index.html` 中项目分析导航的语义结构和无障碍属性。
- 在 `frontend/js/app.js` 中管理当前页面生命周期内的展开状态。
- 在 `frontend/css/style.css` 中实现子菜单缩进、箭头旋转和展开动画。
- 更新前端静态测试，验证默认收起、控件结构和交互代码存在。

## 验收标准

1. 初次打开页面时四个分析入口不可见。
2. 点击“项目分析”后四个入口可见，再次点击后隐藏。
3. 子页面选中状态、页面切换、侧边栏整体折叠和移动端导航正常。
4. 现有前端与后端自动化测试全部通过。
