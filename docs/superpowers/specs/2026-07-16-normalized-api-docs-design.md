# 规范化接口文档设计

## 目标

将项目分析中的接口文档从简单路由清单升级为可阅读、可核对的接口说明，只输出 GET、POST、PUT、DELETE，并统一展示接口名称、请求路径、请求方式、请求参数、请求样例、响应数据格式、备注和源码位置。

## 分析范围

- Spring MVC：识别 `@RestController` 或带 `@ResponseBody` 的接口，读取映射注解、方法签名、`@PathVariable`、`@RequestParam`、`@RequestBody`、DTO/Entity 字段和返回类型。
- FastAPI：保留已识别路由，按同一模板输出；无法从当前静态数据确认的参数和响应明确标注。
- 过滤 PATCH、HEAD、OPTIONS、ANY 以及非 REST 的页面控制器，避免把后台页面 `/` 当作 JSON API。

## 输出规则

- 章节使用 `1.1`、`1.2` 顺序编号。
- 接口名称优先读取 Swagger 注解或方法 Javadoc；没有说明时根据处理器和 HTTP 方法生成稳定名称。
- 路径参数使用示例值 `1`，查询参数使用可读示例值，请求体根据 DTO 字段生成 JSON。
- 响应类型可解析时生成 JSON；无法解析时显示“静态分析未确认具体响应结构”。
- 每项保留源码文件和行号，所有推断结果都可回查。

## 兼容性

不修改数据库表和扫描流程。已有项目只需重新生成接口文档，不需要重新克隆或重新扫描。
