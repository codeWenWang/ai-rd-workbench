# 研发知识工作台前端重设计

## 目标

在不增加构建工具和登录流程的前提下，把现有页面重构为接近现代 AI 对话产品的信息结构，同时修复记忆乱码、候选拒绝不消失、会话命名和输入区遮挡问题。

## 信息结构

- 左侧栏集中品牌、新建对话、知识库、个人记忆、运行状态、中文 API、主题切换和会话历史。
- 会话历史占用侧栏剩余高度并独立滚动；每条会话提供重命名和删除操作。
- 侧栏桌面端可折叠为图标栏，移动端作为覆盖式抽屉。
- 主区只渲染当前功能视图，不再额外保留第二列会话面板。

## 对话体验

- 消息正文限制最大宽度，用户消息显示为右侧圆角气泡，助手消息显示为左侧正文。
- 引用显示为可换行的小型圆角按钮，点击后打开来源抽屉。
- 编辑区固定在主区底部的安全区域内，采用大圆角输入容器，不与悬浮元素或视口边缘重叠。
- 首条用户消息自动生成简短会话标题，用户仍可手动重命名。

## 主题与语言

- 使用 CSS 变量实现浅色和深色主题，并把选择保存在 localStorage。
- 业务页面、状态、错误文案和 API 概览以中文为主。
- `/docs` 提供中文 API 概览，`/openapi.json` 保留机器可读接口定义。

## 数据与错误处理

- 删除当前确认的乱码记忆，同时从 Pinecone 删除对应向量。
- 待确认列表只展示 `pending`，拒绝成功后立即从本地列表移除，再后台刷新。
- 删除当前会话后自动选择下一条；无会话时显示空状态。

## 验证

- Node 测试覆盖布局结构、主题入口、模块缓存版本和候选过滤。
- Pytest 覆盖自动会话命名和中文文档页面。
- Chrome 无缓存实例验证 1440x900、390x844 的浅色和深色截图。
