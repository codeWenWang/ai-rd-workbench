# 项目对话历史折叠分组设计

## 目标

让通用对话和不同项目的对话在“最近对话”中清晰分区，避免通用工作区下所有记录平铺混杂。

## 展示规则

- 最近对话始终读取全部工作区和全部项目的历史。
- 没有 `project_id` 的记录显示在“通用对话”标题下，保持直接可见。
- 每个有对话记录的项目生成一个按钮式折叠分组，显示项目名称、对话数量和方向箭头。
- 项目分组首次加载默认收起；当前正在查看的对话所属项目自动展开。
- 用户手动展开或收起分组的状态在当前页面生命周期内保留，不写入本地存储。
- 新建项目对话后，该对话加入对应项目分组并自动展开。

## 工作区切换

- 切换到通用工作区时，主对话区域优先进入最近的通用对话。
- 切换到某个项目时，主对话区域优先进入该项目最近的对话。
- 如果目标工作区没有历史记录，主区域显示新对话状态，但侧边栏仍展示其他分组。

## 实现范围

- 前端始终调用未带项目过滤条件的对话列表接口。
- 新增纯函数负责按 `project_id` 分组，便于独立测试。
- 项目模块提供按 ID 获取项目名称的只读方法。
- 对话模块负责折叠状态、渲染、选择、删除后的同组回退。
- CSS 使用自适应高度折叠，不限制项目内对话数量。

## 验收标准

1. 通用对话和测试项目对话不再平铺混合。
2. “测试项目”显示为默认收起的折叠分组，并显示对话数量。
3. 点击项目分组可展开和收起，对话的重命名、删除和进入功能正常。
4. 在项目中新建对话后，记录自动出现在对应分组。
5. 切换工作区只影响主区域默认选择，不隐藏侧边栏中的其他分组。
6. 桌面端、移动端和现有自动化测试全部通过。
