# 模型对比与模型管理优化设计

## 目标

解决模型对比结果脱离对话流、显示内部 UUID，以及模型配置只能新增不能维护的问题。

## 模型对比交互

- 对比问题作为普通用户消息进入当前会话。
- 两个模型的结果作为紧随其后的“模型对比”助手消息显示在消息列表内。
- 对比消息使用一个完整回合容器，桌面端双栏、移动端单栏堆叠。
- 每个结果显示“供应商显示名称 · 模型名称”和耗时，不显示内部模型 UUID。
- 引用和检索警告显示在整个对比回合底部，避免重复。
- 对比问题和结果写入 SQLite；刷新页面或重新进入历史对话后仍能恢复。
- 单个模型失败时只标记对应结果，不影响另一个模型的回答。

## 消息存储

- `messages` 表新增可选 `metadata_json` 字段，旧数据库启动时自动补列。
- 普通消息元数据为空；模型对比助手消息保存 `type=model_comparison` 和结果数组。
- 对比接口接收 `session_id`，负责创建或复用会话、生成首条消息标题并持久化用户问题和对比结果。

## 模型管理

- “模型设置”打开管理抽屉，展示已添加模型的名称、模型标识、供应商类型和 Base URL。
- 管理抽屉支持添加、编辑和删除。
- 编辑支持显示名称、供应商类型、Base URL、模型名称和可选 API Key。
- 编辑时 API Key 留空表示继续使用原密钥；接口响应始终不返回明文密钥。
- 新增、编辑或删除后立即刷新聊天模型选择器和模型网关缓存。

## 验收标准

1. 对比结果位于消息列表中，不再出现在输入框下方。
2. 对比卡片显示可识别的模型名称，不显示 UUID。
3. 刷新页面后对比回合仍存在且布局正确。
4. 模型设置可以新增、重命名、修改配置和删除模型。
5. API Key 不在接口或页面中明文显示，留空编辑不会覆盖旧密钥。
6. 桌面端、移动端和现有自动化测试全部通过。
